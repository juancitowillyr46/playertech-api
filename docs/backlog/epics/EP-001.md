# EP-001 Gestión de Academias

## Objetivo
Permitir administrar las academias registradas dentro de la plataforma.

## Problema que Resuelve
La plataforma debe soportar múltiples academias de manera aislada, garantizando que cada una gestione únicamente su propia información.

## Valor de Negocio
Constituye la base del modelo SaaS multi-tenant.

## Actores
Super Admin

## Dominios Involucrados
Academy

## Reglas de Negocio Relacionadas
* Toda academia posee un identificador único.
* Toda academia tiene un estado operativo.
* Una academia suspendida no puede operar normalmente.
* El Super Admin crea y administra tenants desde la plataforma.
* El tenant sólo puede autogestionar la información de su propia academia.

## Flujo de Negocio
1. Super Admin crea academia.
2. Academia queda activa.
3. La academia puede operar dentro del sistema.

## Historias Candidatas
* Crear academia.
* Consultar academia.
* Actualizar academia.
* Suspender academia.
* Reactivar academia.

## Base Tecnica de Referencia
* `AcademyId` se implementa como Doctrine custom type UUID.
* Los VOs de negocio se mapean como XML embeddables.
* La autogestion tenant usa `PUT /api/v1/academy/me`.
* El alta simplificada de tenant debe tener validacion de correo y activacion posterior.

## Métricas de Éxito
* Academias creadas.
* Academias activas.

## Fuera de Alcance
* Facturación SaaS.
* Gestión de planes.

## Preguntas Abiertas
* ¿Una academia podrá cambiar de plan en el futuro?

## MVP
Sí.
