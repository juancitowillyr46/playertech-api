# EP-001 Gestión de Academias

## Objetivo
Permitir administrar las academias registradas dentro de la plataforma.

## Problema que Resuelve
La plataforma debe soportar múltiples academias de manera aislada, garantizando que cada una gestione únicamente su propia información.

## Valor de Negocio
Constituye la base del modelo SaaS multi-tenant.

## Actores
Super Admin
Owner/Admin de la academia

## Dominios Involucrados
Academy

## Reglas de Negocio Relacionadas
* Toda academia posee un identificador único.
* Toda academia tiene un estado operativo.
* Una academia suspendida no puede operar normalmente.
* El Super Admin crea y administra tenants desde la plataforma.
* El tenant sólo puede autogestionar la información de su propia academia.

## Flujo de Negocio
1. Super Admin crea academia.
2. Academia queda activa.
3. La academia puede operar dentro del sistema.
4. El owner/admin administra sólo su propia academia desde `/api/v1/academy/me`.

## Historias Candidatas
* Crear academia desde plataforma.
* Consultar academias desde plataforma.
* Consultar detalle de academia desde plataforma.
* Actualizar academia desde plataforma.
* Consultar propia academia desde tenant.
* Actualizar propia academia desde tenant.
* Suspender academia desde plataforma.
* Reactivar academia desde plataforma.
* Subir escudo institucional de academia.
* Completar perfil basico de academia con ubicacion y consentimientos legales.
* Registrar informacion fiscal basica de academia sin emitir comprobantes.
* Provisionar tenant desde plataforma con admin inicial, correo de bienvenida y equipo inicial opcional.
* Registrar origen de creacion del tenant.

## Base Tecnica de Referencia
* `AcademyId` se implementa como Doctrine custom type UUID.
* Los VOs de negocio se mapean como XML embeddables.
* La autogestion tenant usa `PUT /api/v1/academy/me`.
* El alta simplificada de tenant debe tener validacion de correo y activacion posterior.
* `Academy` incorpora soft delete (`deleted_at`, `deleted_by`) como base tecnica reutilizable.
* La informacion fiscal basica puede existir como extension de perfil, pero la facturacion y DIAN se mantienen fuera de esta epica.
