# EP-014 Alta de Tenant

## Objetivo
Permitir que un futuro tenant se registre con datos simplificados, active su cuenta por correo y quede habilitado para operar en su propio contexto.

## Problema que Resuelve
El alta de un tenant no debe depender únicamente del flujo administrado por el Super Admin. La plataforma necesita un onboarding propio para acelerar la creación de academias con validación de identidad y activación.

## Valor de Negocio
Reduce fricción en la captación de nuevos clientes y profesionaliza el onboarding del SaaS.

## Actores
Tenant Prospect
Tenant Academy Admin
Super Admin

## Dominios Involucrados
Identity
Academy

## Reglas de Negocio Relacionadas
* El Super Admin sigue pudiendo crear tenants desde la plataforma.
* El tenant puede registrarse con un flujo simplificado.
* El alta del tenant crea la academia y el usuario owner/admin inicial.
* El correo del tenant debe validarse antes de habilitar el acceso completo.
* Un tenant no activado no puede operar como tenant autenticado.

## Flujo de Negocio
1. El tenant ingresa datos simplificados.
2. El sistema crea la academia en estado pendiente.
3. El sistema crea el usuario owner/admin del tenant.
4. El sistema envía correo de activación.
5. El tenant confirma su correo.
6. El sistema activa la cuenta y habilita el acceso.

## Historias Candidatas
* Registrar tenant con datos simplificados.
* Enviar correo de activación del tenant.
* Activar cuenta de tenant desde correo.
* Reenviar correo de activación.

## Base Tecnica de Referencia
* El flujo debe reutilizar `Academy` y `Identity`.
* El registro debe generar `academy_id` y usuario owner/admin asociados.
* El estado inicial del tenant debe impedir acceso operativo hasta activar el correo.
* La mensajeria de activacion debe desacoplarse con `Symfony Messenger`.
* El envio de correo debe apoyarse en `Symfony Mailer`.
* El entorno local de prueba recomendado para correos es `Mailpit` o equivalente.
* El canal de correo debe quedar configurable por entorno para poder usar un proveedor real en produccion y un inbox de pruebas en desarrollo.
