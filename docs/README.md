# Documentation Index

`docs/` contiene la documentacion de producto, backlog, arquitectura y contratos del proyecto.

## Structure

- `docs/product/` visión, roadmap y contexto de negocio.
- `docs/backlog/` épicas, historias y priorización funcional.
- `docs/architecture/` ADR, auditorías, políticas y plantillas de cambio.
- `docs/contracts/` índice de contratos HTTP y sincronización con frontend.

## Usage Rule

- Si una decisión afecta al producto o al backlog, vive en `docs/`.
- Si una decisión afecta al contrato técnico o a la arquitectura, vive en `specs/` o `docs/architecture/` según su naturaleza.
